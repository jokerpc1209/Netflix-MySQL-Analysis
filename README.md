# Netflix 影视内容 MySQL 多维度分析项目
## 项目背景
基于8800+条Netflix真实影视数据，用MySQL完成“数据清洗→结构化存储→多维度分析→性能优化”全流程，挖掘内容类型分布、地区偏好、年度上线趋势等业务洞察，为内容运营提供建议。
## 技术栈
- 数据库：MySQL 8.0（建表、递归CTE、索引优化）
- 数据处理：Python（CSV清洗、日期格式统一）
- 代码托管：GitHub + Kaggle Kernel
## 核心流程
### 1. 数据预处理
- 清洗原始CSV：统一`date_added`为`YYYY-MM-DD`格式，去重、删除空行，生成`netflix_titles_clean.csv`；
- 处理痛点：解决多标签（listed_in）拆分、日期格式不兼容问题。
### 2. MySQL 结构化设计
- 建表：设计13个字段的`netflix_content`表，含主键约束（show_id）、空值处理（director允许空）；
- 导入：用`LOAD DATA INFILE`完成CSV导入，解决本地权限、编码报错等问题。
### 3. 多维度分析（SQL核心）
| 分析维度       | 实现方式                  | 关键结论                                  |
|----------------|---------------------------|-------------------------------------------|
| 类型分布       | GROUP BY + 聚合函数       | 电影占69%，剧集占31%                      |
| 多标签分类     | 递归CTE拆分listed_in      | 国际电影、剧情片、喜剧片是TOP3分类        |
| 地区偏好       | SUBSTRING_INDEX取主地区   | 美国电影占75%，印度电影占比最高（非美国） |
| 年度上线趋势   | YEAR(date_added) + 索引   | 2020年上线1800条（疫情驱动居家需求）      |
### 4. 性能优化
- 问题：无索引时“年度内容统计”查询耗时1.2秒；
- 优化：创建联合索引`idx_release_year_type`；
- 效果：查询耗时降至0.1秒，效率提升90%+。
